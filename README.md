# Быстрый запуск

 1. Клонирование и создание вертуального окружения
```bash
git clone https://github.com/ivanzobenko/Lab11Winx
cd Lab11Winx
python -m venv venv

# Активация:
# Windows: venv\Scripts\activate
# Mac/Linux: source venv/bin/activate

```
2. Установка и настройка
   
```bash

pip install -r requirements.txt

python manage.py migrate

python manage.py runserver
```
Откройте в браузере: http://127.0.0.1:8000/

 # Архитектура (MVC)
 **Используется**  Django ORM + SQLite

Модели (models.py)-Отвечает за структуру данных и взаимодействие с базой данных SQLite 

Post - Хранит статьи блога (заголовок, текст, статус, дата, автор)

Comment - Хранит комментарии к постам

User-Хранит данные пользователей (логин, пароль)

# Сложности, с которыми мы столкнулись
В процессе разработки возникло несколько типичных сложностей. Во-первых, на начальном этапе были трудности с настройкой структуры Django-проекта и правильным подключением приложений, особенно при работе с пользовательской моделью и миграциями базы данных. Иногда после изменения моделей требовалось пересоздавать миграции, чтобы база данных корректно обновлялась.
Во-вторых, при командной работе с использованием Git Flow возникали конфликты при слиянии веток, особенно когда несколько участников одновременно работали с одними и теми же файлами. Эти проблемы решались за счёт чёткого разделения зон ответственности и аккуратного разрешения конфликтов.
Также определённую сложность вызвала связка бэкенда и фронтенда — необходимо было согласовать модели, представления и шаблоны, чтобы данные корректно передавались и отображались на страницах.
В итоге все сложности были решены, и проект удалось успешно интегрировать в единую рабочую версию.
